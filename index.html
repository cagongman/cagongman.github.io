프로젝트 개요

본 프로젝트는 실제 반려동물이 주인의 신체적, 정신적 이점을 가져다 준다는 내용을 받침으로 가상 반려동물 또한 신체 활동에 긍정적인 영향을 미치는지 파악하기 위해 설계되었습니다.

이를 위해 가상 반려견과 함께 홈트레이닝을 하는 피트니스 버디 프로그램을 개발하였으며 사용자 실험을 통해 가상 반려견이 운동 경험에 미치는 영향을 분석하였습니다.

상세 내용

게임 모드

본 애플리케이션은 세가지의 운동 모드를 지원합니다.

  • - No Virtual Dog (NVD)
    • 가상 강아지 없이 오직 운동 영상이 재생되는 가상 스크린 만이 존재하는 모드입니다.
  • - Capricious Virtual Dog (CVD)
    • 무작위로 행동하는 가상 강아지와 함께 운동하는 모드입니다.
  • - Interactive Virtual Dog (IVD)
    • 사용자가 운동 동작을 수행할때마다 동작을 따라하는 가상 강아지와 함께 운동하는 모드입니다.

또한 본 애플리케이션은 2인 멀티플레이를 제공합니다. 친구와 함께 세가지 모드를 즐길 수 있습니다. 멀티플레이 환경에서 ‘IVD’ 모드를 실행할 경우 강아지는 두 사용자의 행동이 일치할 경우에만 동작을 따라하게 됩니다.



게임 환경

게임 모드를 선택한 후 사용자는 옆에 가상 준비 버튼과 앞에 가상 스크린을 확인할 수 있습니다.

가상 화면에는 Reach(팔 뻗기), Squat(스쿼트), Punch(펀치), Stretching(스트레칭) 네 가지 동작으로 구성된 1분 30초 길이의 운동 영상이 재생됩니다.



변덕스러운 가상 강아지 (CVD 모드)

실제 반려견처럼 행동하도록 만들기 위해서 강아지 모션 애니메이션 클립을 세개의 부모 상태와 해당 자식 행동으로 구성하였습니다. 세 개의 부모 상태는 기본 상태 (서있는 상태), 앉아있는 상태, 누워있는 상태 세가지 이며 각 상태에서 강아지가 취할만한 행동인 햝기, 긁기, 털기 등의 자식 행동을 설정하였습니다.

마지막으로 5초에서 10초 사이의 임의의 간격으로 가상 강아지가 무작위 행동을 취하도록 구현하였습니다.



따라쟁이 가상 강아지 (IVD 모드)

먼저 사용자의 운동 동작을 인식하기 위해 Meta Quest 3에서 제공하는 IOBT(Inside-Out Body Tracking) 기술과 Generative Leg 기술을 활용했습니다. 이를 통해 사용자의 전신의 3D 관절 좌표를 얻을 수 있습니다.

추출한 3D 관절 좌표를 통해 아래와 같은 조건을 세워 각 동작을 인식했습니다.

  • Reach(뻗기): 손목 관절의 y 좌표가 머리보다 높아야 하며, 팔꿈치 각도가 150°에서 210° 사이인지 확인하여 팔이 완전히 뻗어졌는지 확인합니다. 어깨 관절에서 손목까지의 벡터 방향을 사용하여 동작이 왼쪽인지 오른쪽인지 결정합니다.
  • Squat(스쿼트): 발목, 무릎, 골반 좌표를 사용하여 무릎 각도가 100° 미만이면 앉은 자세로, 100° 이상이면 서 있는 자세로 인식합니다.
  • Punch(펀치): Reach 동작과 유사하게 팔이 완전히 뻗어졌는지 확인하고, 손목의 x 좌표가 반대쪽 어깨 관절의 x 좌표보다 앞에 있는지 확인하며, 팔이 앞으로 평행하게 뻗어 있는지 및 어느 방향을 향하고 있는지를 확인합니다.
  • Stretch(스트레칭): 손목 관절의 y 좌표가 머리보다 높은지 확인합니다. 골반, 어깨, 손목 관절의 좌표를 사용하여 내부 어깨 각도가 160° 미만인지 확인하여 팔이 올바르게 들어올려졌는지 및 상체가 충분히 기울어졌는지 확인합니다.

각 동작의 조건에 따라 운동 동작이 인식되면, 가상 강아지는 해당 동작과 방향에 맞는 애니메이션을 재생합니다.

2초 이상 사용자 동작이 인식되지 않으면 가상 강아지는 짖으면서 운동을 재촉합니다.



멀티 플레이 구현

AWS EC2 서비스를 사용하여 서버를 구축하고 소켓통신을 통해 두 클라이언트 간의 상호작용을 가능하게 하였습니다.

각 클라이언트는 준비 상태, 수행한 운동동작 데이터를 serialization한 패킷을 준비 버튼을 눌렀을때와 운동 동작을 수행할때마다 서버에 전송합니다.

서버는 두 클라이언트의 준비 상태가 모두 true일때 두 클라이언트에게 운동 시작을 알리고 broadcasting을 통해 두 클라이언트의 운동 영상을 재생시킵니다. 또한 운동 중에는 두 클라이언트로부터 수신한 운동 동작이 동일한 경우 broadcasting을 통해 두 클라이언트의 가상 강아지의 애니메이션을 재생시킵니다.



사용자 실험

사용자 실험을 통해 가상 강아지의 효과를 분석하였습니다.

실험당 참가자는 두명씩 참가하여 실험을 진행했습니다. 오리엔테이션을 진행하고 참가자분들은 본 시스템에서 제공하는 모든 게임 모드를 한번씩 플레이하도록 하였습니다.

1인 플레이(Individual)의 경우에는 파티션으로 두 참가자의 공간을 나눠서 진행했습니다.

참가자들은 각 게임 모드 종료 후 운동 경험(재미, 참여도, 몰입도, 동기부여), 가상 강아지에 대한 인식(애정도, 사회적 존재감)에 관한 설문조사를 수행하였습니다.



결과

총 24명(남: 13, 여: 11; 나이: M = 23.63, SD = 5.49)의 참가자들을 통해 설문 결과를 얻었습니다.

이원 반복측정 분산분석(2-way ANOVA)를 통해 데이터의 유의미성을 분석했고, 그 결과 아래와 같은 점을 알게 되었습니다.

  • - 가상 강아지의 존재가 사용자들의 운동 경험을 향상시킨다. (NVD에서 CVD로 16.34% 향상)
  • - 가상 강아지와의 상호작용이 포함될 경우 사용자들의 운동 경험이 더욱 향상된다. (CVD에서 IVD로 23.03% 향상)
  • - 상호작용 요소로 인해 가상 강아지 인식이 더욱 크게 긍정적으로 변한다. (CVD에서 IVD로 46.87% 향상)
  • - 운동 경험 및 가상 강아지 인식에 있어서 개인과 그룹 운동 간에는 차이가 없다.

Accident Detection


프로젝트 개요

건설 현장에서 혼자 작업하던 작업자가 사고를 당하면 즉각적인 구조나 응급처치가 어려운 경우가 많습니다.

이런 문제를 해결하기 위해, 본 프로젝트에서는 가상 에이전트를 사용해 추락, 충돌 사고를 재현하고, 사고 시 발생하는 머리 움직임 데이터를 수집했습니다.

이 데이터를 바탕으로 사고를 자동으로 감지하는 인공지능 모델을 개발했습니다.

에이전트 설정

충돌 및 추락 상황을 연출하기 위해서 유니티 게임엔진의 Ragdoll physics 기능을 사용하였습니다.

기본(Idle) 상태의 에이전트에 충돌을 감지하는 콜라이더를 설정하여 콜라이더에 충돌 객체가 감지될 경우 기본 상태 에이전트의 각 관절의 위치와 회전 값을 ragdoll physics가 적용된 에이전트에 동일하게 적용한 후 교체시켰습니다.



시뮬레이션 구현

본 프로젝트에서는 자재 낙하로 인한 충돌 사고와 고층 높이에서 떨어지는 추락 사고를 구현하였습니다.

충돌 사고의 경우 충돌 객체 (벽돌, 파이프, 철 기둥, 철망 펜스) 4 가지를 사용하였습니다. 각각의 초기 무개(5kg, 40kg, 60kg, 15kg)를 설정한 후 캐릭터의 일정 높이 위에서 낙하 시켰습니다.

추락 사고의 경우 에이전트를 임시 발판과 사다리 위에 위치시킨 후 흉부 쪽에 AddForce() 함수를 사용하여 impulse 모드로 힘을 가해 떨어뜨렸습니다.

일반 상황의 경우 30개의 일상 생활 관련 모션 에니메이션 클립을 사용하였습니다.



데이터 생성

학습 데이터를 생성하기 위해 매 시뮬레이션에서 특정 매개변수를 무작위로 설정하여 여러 시나리오 진행하였습니다.

  • 충돌 객체: 충돌 객체의 높이와 무게는 기존 값에 1.0에서 2.0 사이의 무작위 값을 곱하였습니다. scale은 무게와 동일 값을 곱해주었습니다.
  • 에이전트: 충돌, 추락, 일반 상황에서 모두 에이전트의 y축을 기준으로 회전 무작위로 회전시킴
  • 일반 상황 애니메이션: 애니메이션의 재생 속도를 0.5 ~ 1.5 사이의 무작위로 설정하였습니다.

데이터수집은 60fps로 고정한 후 매 프레임마다 에이전트 머리의 position, rotation, velocity, angular velocity 값을 기록했습니다.

충돌 객체 및 낙하 유형 별로 각 100회씩 총 600회의 시뮬레이션을 진행하였고 그 중 연속된 30 프레임을 무작위로 1000개 추출하였습니다.

일반 상황 데이터 또한 각 에니메이션 클립 별로 10회씩 총 300회 시뮬레이션을 진행한 후 연속된 30 프레임 1000개를 추출했습니다.


네트워크

사고 감지 모델의 입력은 연속된 30프레임으로 구성되며, 각 프레임은 정규화 되어 13 차원으로 이루어져 있습니다. 머리 움직임 데이터의 순차적 특성을 고려하여, conv1d-LSTM 구조의 신경망을 채택하였습니다.

최종적으로 SoftMax 함수를 사용하여 위험 상황과 일반 상황의 두 가지 클래스에 대한 확률 값을 추출했습니다.

활성화 함수로는 ReLU를 사용했으며, 손실 함수와 최적화 알고리즘으로는 각각 Cross-Entropy와 Adam을 사용했습니다.



실험 및 결과

일반 상황 데이터 1000개, 사고 상황 데이터 1000개를 가지고 아래와 같이 하이퍼파라미터를 설정한 후 학습을 진행한 결과 테스트 데이터셋 정확도 94.95%를 얻었습니다.

  • 데이터셋 비율: Test 20%, Validation 16%, Train 64%
  • Learning rate: 0.001
  • Epochs: 20
  • Batch size: 64

유니티의 Barracuda 라이브러리를 사용하여 학습된 모델을 실행하였습니다. 구현한 시뮬레이션 환경에서 충돌 및 추락사고를 정확히 감지하는 것을 확인할 수 있었습니다.

SOS 버튼

SOS 버튼 상호작용

사용자가 SOS 버튼을 활성화하고 있다고 인지시키기 위해 세가지 피드백을 제공하였습니다.

  • 최대 게이지 표시: 버튼 크기의 2배인 빨간색 원이 SOS 버튼 뒤에 나타납니다.
  • 버튼 사이즈 변화: SOS 버튼이 점차 커집니다. 버튼의 크기는 3초 동안 첫 번째 효과로 생성된 원의 크기만큼 증가합니다.
  • 원형 타이머 생성: 작업자가 바라보고 있는 지점에 원형 타이머가 생성되며 긴급 요청이 실행될 때까지 남은 시간이 숫자로 표시되고 게이지바가 채워지게 됩니다.


구조 요청 기능 활성화

SOS 버튼이 활성화 되면 배경이 빨간색으로 바뀌고 사용자는 재확인 창을 제외한 다른 구성 요소와 상호 작용할 수 없게 됩니다.

"예" 버튼과 "아니오" 버튼은 시선을 통한 상호작용의 오차를 최대한 방지하기 위해 양쪽 끝에 배치하였습니다.

SOS 버튼 이외의 모든 버튼들 또한 시선과 터치를 통해 상호작용이 가능합니다.

상세 내용

치아 보철물 오브젝트 재정렬

Object 파일을 처음 import할 시 보철물의 방향이 제각각이기 때문에

Import 후에 보철물이 정면을 주시하도록 방향을 수정하는 작업이 필요했습니다.


Face mesh 생성

ARCore에서 제공하는 Augmented Face API를 사용하여 환자의 얼굴을 인식한 후

468개의 vertices를 가지는 Face mesh를 생성합니다.


매칭 마커 선택

Face mesh에 보철물 오브젝트를 적합한 위치에 매칭시키기 위해 각각에 대응되는 마커를 3개 이상 선택합니다.


치아 보철물 위치 최적화

Face mesh와 보철물 object에서 선택한 각각의 대응되는 마커들을 사용하여 보철물 오브젝트를 위치시킵니다.

각각의 대응되는 마커 간의 거리가 최소가 되도록 보철물 오브젝트의 위치를 최적화시켰습니다.

최적화방법으로는 Levenberg–Marquardt algorithm(Alglib)을 적용하였습니다.

Face occluder material을 사용하여 치아와 잇몸 부분을 제외한 나머지부분에 대해서 오브젝트를 제거하였습니다.

우측 하단의 조작버튼으로 위치와 각도를 미세하게 조정이 가능합니다.

히트맵 2차 보간

1차 보간 후에도 아직 빈 곳이 너무 많아 확인해 본 결과 결측치가 아닌 매우 작은 값으로 잘못 채워져 있는 것을 확인했습니다.
