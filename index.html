AR Fitness Dog


프로젝트 개요

실제 반려동물이 주인의 신체적, 정신적, 사회적 이점을 가져다 준다는 연구 결과가 있습니다. 이 프로젝트는 실제 반려동물의 여러 이점들 중 신체적 이점에 초점을 두고 가상 반려동물 또한 신체 활동에 긍정적인 영향을 미치는지 파악하기 위해 설계되었습니다.

이를 위해 가상 반려견과 함께 홈트레이닝을 하는 피트니스 버디 시스템을 개발하였으며 사용자 실험을 통해 가상 반려견이 운동 경험에 미치는 영향을 분석하였습니다.

상세 내용

게임 모드

본 애플리케이션은 세가지의 운동 모드를 지원합니다.

  • - No Virtual Dog (NVD)
    • 이 모드에서는 가상 강아지가 나오지 않고, 오직 운동 동작을 보여주는 영상만 재생됩니다.
  • - Capricious Virtual Dog (CVD)
    • 이 모드에서는 추가로 무작위로 행동하는 가상 강아지가 등장합니다.
  • - Interactive Virtual Dog (IVD)
    • 이 모드에서는 가상 강아지가 사용자의 운동 동작을 따라합니다.

또한 본 시스템은 2인 멀티플레이를 제공합니다. 친구와 함께 세가지 모드를 모두 즐길 수 있습니다.

멀티플레이 환경에서 ‘IVD’ 모드를 실행할 경우 두 사용자의 행동이 일치할 경우에만 강아지가 동작을 따라하게 됩니다.



게임 환경

게임 모드를 선택한 후 사용자의 오른쪽에는 가상 준비 버튼, 정면에는 가상 스크린이 나타납니다.

사용자는 준비 버튼을 눌러 운동을 시작할 수 있습니다. 가상 화면에는 팔 뻗기(Reach), 스쿼트(Squat), 펀치(Punch), 스트레칭(Stretching) 네 가지 동작으로 구성된 1분 30초 길이의 운동 영상이 재생됩니다.



변덕스러운 가상 강아지 (CVD 모드)

강아지의 자연스러운 행동을 구현하기 위해 다음과 같은 애니메이션 시스템을 설계했습니다:

1. 기본 자세 3가지

  • - 서있기
  • - 앉기
  • - 눕기

2. 각 기본 자세에 연결된 세부 행동들

  • - 예: 햝기, 긁기, 털기 등

이러한 구조를 통해 가상의 반려견이 실제 강아지처럼 다양한 동작을 수행할 수 있습니다.

더불어, 강아지의 행동이 예측 불가능하고 생동감 있게 보이도록 5~10초 간격으로 무작위 행동을 선택하여 실행하였습니다.



따라쟁이 가상 강아지 (IVD 모드)

Reach

Squat

Punch

Stretch

먼저 사용자의 운동 동작을 인식하기 위해 Meta Quest 3에서 제공하는 IOBT(Inside-Out Body Tracking) 기술과 Generative Leg 기술을 활용하여 사용자 전신의 3D 관절 좌표를 추출하였습니다.

추출한 좌표들을 기반으로 팔, 다리 등 관절 각도에 조건을 설정하여 각 운동 동작을 인식했습니다.



멀티 플레이 구현

AWS EC2 서비스를 사용하여 서버를 구축하고 소켓통신을 통해 두 클라이언트 간의 상호작용을 가능하게 하였습니다.

각 클라이언트는 준비 상태, 수행한 운동동작 데이터를 준비 버튼을 눌렀을때와 운동 동작을 수행할때마다 서버에 전송합니다.

서버는 두 클라이언트의 준비 상태가 모두 true일때 두 클라이언트에게 운동 시작을 알리고 운동 영상을 동시에 재생시킵니다. 또한 운동 중에는 두 클라이언트로부터 수신한 운동 동작이 동일한 경우 가상 강아지의 해당 운동 동작 애니메이션을 재생시킵니다.



사용자 실험

사용자 실험을 통해 가상 강아지의 효과를 분석하였습니다.

실험당 참가자는 두명씩 참가하여 실험을 진행했습니다. 오리엔테이션을 진행하고 참가자분들은 본 시스템에서 제공하는 모든 게임 모드를 한번씩 플레이하도록 하였습니다.

1인 플레이(Individual)의 경우에는 파티션으로 두 참가자의 공간을 나눠서 진행했습니다.

참가자들은 각 게임 모드 종료 후 운동 경험(재미, 참여도, 몰입도, 동기부여), 가상 강아지에 대한 인식(애정도, 사회적 존재감)에 관한 설문조사를 수행하였습니다.



결과

총 24명(남: 13, 여: 11; 나이: M = 23.63, SD = 5.49)의 참가자들을 통해 설문 결과를 얻었습니다.

이원 반복측정 분산분석(2-way ANOVA)를 통해 데이터의 유의미성을 분석했고, 그 결과 아래와 같은 점을 알게 되었습니다.

  • - 가상 강아지의 존재가 사용자들의 운동 경험을 향상시킨다. (NVD에서 CVD로 16.34% 향상)
  • - 가상 강아지와의 상호작용이 포함될 경우 사용자들의 운동 경험이 더욱 향상된다. (CVD에서 IVD로 23.03% 향상)
  • - 상호작용 요소로 인해 가상 강아지에 대한 인식이 더욱 크게 긍정적으로 변한다. (CVD에서 IVD로 46.87% 향상)
  • - 운동 경험 및 가상 강아지에 대한 인식에 있어서 개인과 그룹 운동 간에는 차이가 없다.


OREH : '황기환' 그의 귀환 야외 방탈출 게임


프로젝트 개요

100년만에 고국 품으로 돌아온 애국지사 "황기환", 그의 이야기를 담은 야외 방탈출 게임을 개발하였습니다.

플레이어는 국가유공자의 조력자가 되어 게임을 진행하게 됩니다. 장소는 덕수궁에서 진행되며 같이 제공되는 키트와 함께 더욱 몰입감있고 현장감 넘치는 경험을 선사합니다.

본 게임의 목적은 MZ세대/청소년들에게 국가유공자들의 업적에 대해 알리기 위함입니다.

퀴즈의 답은 편지 안의 미션지를 활용하여 풀어나가야 합니다.

프로젝트 개요

건설현장 지원 시스템의 후속 연구로 사고 발생 시 작업자의 사고를 자동으로 인식하고 구조 요청을 보내기 위한 기능을 개발했습니다.

이를 위해, 가상 스턴트맨을 활용하여 추락이나 충돌 사고를 시뮬레이션하고, 이로부터 머리 움직임 데이터를 추출했습니다. 추출한 데이터를 기반으로 LSTM(Long Short-Term Memory) 네트워크를 이용한 사고 감지 모델을 구현하고 학습시켰습니다.

학습된 모델은 ONNX 형식으로 변환하고, 시뮬레이션 환경에 적용하여 테스트를 수행했습니다.

상세 내용

가상 스턴트맨 설정

충돌 및 추락 상황을 연출하기 위해서 유니티 게임엔진의 Ragdoll physics 기능을 사용하였습니다.

기본(Idle) 상태의 에이전트에 충돌을 감지하는 콜라이더를 설정하여 콜라이더에 충돌 객체가 감지될 경우 기본 상태 에이전트의 각 관절의 위치와 회전 값을 ragdoll physics가 적용된 에이전트에 동일하게 적용한 후 교체시켰습니다.



시뮬레이션 구현

본 프로젝트에서는 자재 낙하로 인한 충돌 사고와 고층 높이에서 떨어지는 추락 사고를 구현하였습니다.

충돌 사고를 재현하기 위해 충돌 객체: 벽돌(brick), 파이프(pipe), 철 기둥(steel pillar), 철망 펜스(wire fence) 4 가지를 캐릭터의 일정 높이 위에서 낙하 시켰습니다. 각각의 초기 무개는 5kg, 40kg, 60kg, 15kg로 설정하였습니다.

추락 사고를 재현하기 위해 에이전트를 임시 발판과 사다리 위에 위치시킨 후, AddForce() 함수를 사용해 흉부 쪽으로 힘을 가해 밀쳐내는 방식을 사용했습니다. 이때 힘은 'impulse' 모드로 적용했습니다.

일반 상황의 경우 30개의 일상 생활 관련 모션 에니메이션 클립을 사용하였습니다.



데이터 생성

학습 데이터를 생성하기 위해 매 시뮬레이션에서 특정 매개변수를 무작위로 설정하여 여러 시나리오 진행하였습니다.

  • 충돌 객체: 충돌 객체의 높이와 무게는 기존 값에 1.0에서 2.0 사이의 무작위 값을 곱하였습니다. scale은 무게와 동일 값을 곱해주었습니다.
  • 에이전트: 충돌, 추락, 일반 상황 모두에서 에이전트를 y축을 기준으로 무작위로 회전시켰습니다.
  • 일반 상황 애니메이션: 애니메이션의 재생 속도를 0.5 ~ 1.5 사이의 무작위로 설정하였습니다.

머리 움직임 데이터는 60fps로 고정한 후 매 프레임마다 에이전트 머리의 position, rotation, velocity, angular velocity 값을 수집했습니다.

충돌 객체 및 낙하 유형 별로 각 100회씩 총 600회의 시뮬레이션을 진행하였고 그 중 연속된 30 프레임을 무작위로 1000개 추출하였습니다.

일반 상황 데이터 또한 각 에니메이션 클립 별로 10회씩 총 300회 시뮬레이션을 진행한 후 연속된 30 프레임 1000개를 추출했습니다.


네트워크

사고 감지 모델의 입력은 연속된 30프레임으로 구성되며, 각 프레임은 정규화 되어 13 차원으로 이루어져 있습니다. 머리 움직임 데이터의 순차적 특성을 고려하여, conv1d-LSTM 구조의 신경망을 채택하였습니다.

최종적으로 SoftMax 함수를 사용하여 위험 상황과 일반 상황의 두 가지 클래스에 대한 확률 값을 추출했습니다.

활성화 함수로는 ReLU를 사용했으며, 손실 함수와 최적화 알고리즘으로는 각각 Cross-Entropy와 Adam을 사용했습니다.



실험 및 결과

일반 상황 데이터 1000개, 사고 상황 데이터 1000개를 가지고 아래와 같이 하이퍼파라미터를 설정한 후 학습을 진행한 결과 테스트 데이터셋 정확도 94.95%를 얻었습니다.

  • 데이터셋 비율: Test 20%, Validation 16%, Train 64%
  • Learning rate: 0.001
  • Epochs: 20
  • Batch size: 64

개발된 시스템의 실제 성능을 검증하기 위해 유니티 엔진의 신경망 추론 라이브러리인 Barracuda를 활용하여 학습된 LSTM 모델을 통합하였습니다.

테스트 결과, 구현된 시뮬레이션 상에서 충돌 사고와 추락 사고를 성공적으로 식별해내는 것을 확인했습니다.

SOS 버튼

SOS 구조 요청

사용자가 SOS 버튼 및 버튼을 활성화하고 있다고 인지시키기 위해 세가지 피드백을 제공하였습니다.

  • 최대 게이지 표시: 버튼 크기의 2배인 빨간색 원이 SOS 버튼 뒤에 나타납니다.
  • 버튼 사이즈 변화: SOS 버튼이 점차 커집니다. 버튼의 크기는 3초 동안 첫 번째 효과로 생성된 원의 크기만큼 증가합니다.
  • 원형 타이머 생성: 작업자가 바라보고 있는 지점에 원형 타이머가 생성되며 긴급 요청이 실행될 때까지 남은 시간이 숫자로 표시되고 게이지바가 채워지게 됩니다.

SOS 버튼이 활성화 되면 배경이 빨간색으로 바뀌고 사용자는 재확인 창을 제외한 다른 구성 요소와 상호 작용할 수 없게 됩니다.

"예" 버튼과 "아니오" 버튼은 시선을 통한 상호작용의 오차를 최대한 방지하기 위해 양쪽 끝에 배치하였습니다.

SOS 버튼 및 모든 버튼들은 시선과 터치를 통해 상호작용이 가능합니다.

Import 후에 보철물이 정면을 주시하도록 방향을 수정하는 작업이 필요했습니다.


Face mesh 생성

ARCore에서 제공하는 Augmented Face API를 사용하여 환자의 얼굴을 인식한 후

468개의 vertices를 가지는 Face mesh를 생성합니다.


매칭 마커 선택

Face mesh에 보철물 오브젝트를 적합한 위치에 매칭시키기 위해 각각에 대응되는 마커를 3개 이상 선택합니다.


치아 보철물 위치 최적화

Face mesh와 보철물 object에서 선택한 각각의 대응되는 마커들을 사용하여 보철물 오브젝트를 위치시킵니다.

각각의 대응되는 마커 간의 거리가 최소가 되도록 보철물 오브젝트의 위치를 최적화시켰습니다.

최적화방법으로는 Levenberg–Marquardt algorithm(Alglib)을 적용하였습니다.

Face occluder material을 사용하여 치아와 잇몸 부분을 제외한 나머지부분에 대해서 오브젝트를 제거하였습니다.

우측 하단의 조작버튼으로 위치와 각도를 미세하게 조정이 가능합니다.



파도를 이용한 원격 감지


프로젝트 개요

파도의 높이는 기후 변화, 태풍, 지진 등에 의해 영향을 받을 수 있으며, 이러한 자연 현상의 발생 빈도와 강도를 모니터링하는 중요한 지표가 됩니다.

이 프로젝트는 방파제에 부딪치는 파도의 높이를 자동으로 측정하기 위해 제안되었습니다. 이를 위해 실제 강릉 주문진 방파제의 3D 모델을 사용하여 모니터링 카메라 영상의 height map을 추출하는 작업이 진행되었습니다.

상세 내용

방파제 높이 값 추출

Unity3D에서의 카메라 view에 나타나는 방파제 object의 위치와 촬영한 영상에서의 방파제 위치를 대략적으로 일치시킵니다.

모든 픽셀좌표에 대해서 ray를 쏘아 방파제 object에 충돌한 3D 좌표의 y값(height)을 추출하여 영상과 같은 크기의 height map을 생성하였습니다.


Height Map 1차 보간

Height map을 시각화해본 결과 ray casting에 실패한 부분들이 많았습니다.

실패한 부분들을 채우고자 Scipy 라이브러리의 interpolate.griddata() 함수를 활용하여 결측치를 보간하였습니다.


Height Map 2차 보간

1차 보간 후에도 아직 빈 곳이 너무 많아 확인해 본 결과 결측치가 아닌 매우 작은 값으로 잘못 채워져 있는 것을 확인했습니다.

이상치를 제거한 후 다시 값을 보간하였습니다.

프로젝트 개요

공공데이터 활용 경진대회 "우수상"을 수상하였습니다.

거리에 무분별하게 버려진 쓰레기와 지정된 수거함을 찾기 어려운 문제를 해결하기 위해, 쓰레기통 위치와 올바른 폐기법을 알려주는 서비스 '에코빈'을 개발하게 되었습니다.

이 서비스는 보상 제도를 통해 시민들이 올바르게 쓰레기를 처리하도록 장려합니다.

상세 내용

공공데이터포털에서 대구 중구 쓰레기통 설치 현황 데이터와 폐건전지, 폐형광등 분리수거함 위치 데이터를 가져와 사용했습니다.

위의 데이터는 지번 주소 또는 장소 이름으로 저장되어 있기때문에 Google Geocode API를 사용하여 경도, 위도 좌표로 변환해 주었습니다.

추첨공 획득

MapKit API를 통해 생성된 지도에 변환한 좌표(경도, 위도)를 건전지, 형광등, 일반쓰레기 카테고리별로 나누어 marker를 추가해 시각화 하였습니다.

쓰레기를 버린 쓰레기통 위치의 마커를 클릭하여 "추첨 공"을 받을 수 있습니다.

추첨 공: 상품 추첨 행사에 사용되는 응모권입니다.
